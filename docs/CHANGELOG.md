# Changelog

## 0.5.1 — 2026-07-15

### Автопроверка обновлений
- Приложение раз в сутки (плюс через ~минуту после запуска) опрашивает GitHub
  Releases (условные запросы с ETag; ошибки сети молча ждут следующего тика).
- При новой версии: одно уведомление + пункт «⬆ Обновить до X» в трей-меню.
  По клику — самообновление: скачивает .dmg релиза, проверяет (bundle id,
  версия, целостность подписи), подменяет /Applications/ghbdtn.app и
  перезапускается. Грант Accessibility сохраняется (та же подпись), Gatekeeper
  не вмешивается (загрузка без карантина). Любая ошибка — фолбэк на страницу
  релиза.
- Ручная проверка: пункт «Проверить обновления» в меню. Тумблер автопроверки —
  в Настройках (по умолчанию включена). Dev-сборки вне /Applications никогда
  не перезаписываются.
- Диагностика: `ghbdtn --updatecheck` (с `GHBDTN_FAKE_VERSION=0.4.0` — прогон
  сценария «есть обновление»).

## 0.5.0 — 2026-07-15

Большой релиз надёжности конверсии. Исправлены все баги из аудита
[audit-conversion-bugs-2026-07-15.md](audit-conversion-bugs-2026-07-15.md);
цифры «до → после» — из прогонов пробника на реальном движке
(balanced, floor=3, корпуса tools/domain-corpora + /usr/share/dict/words).

### Конверсия в неправильную сторону (BUG-2)
- Порог «n-gram ручается сам» (percentile > 0.05): правильно набранные
  заимствования («энвайрменте», «алерт», «деплой», «берч») больше не уходят в
  облако и не конвертируются. Эскалация правильных слов в AI:
  **48,6% → 3,5%** (RU-домен), 69,8% → 15,6% (EN-словарь), 29,8% → 5,1% (EN-домен).
- Гейт «AI-вердикт обязан совпадать с раскладочным твином»: облачная модель
  больше не может «перевести» слово («фетч» → "fetch" отклоняется, потому что
  твин — "atnx").
- Второй AI-гейт (по итогам адверсариальной проверки): вердикт отклоняется,
  когда ни один локальный сигнал не может судить ни одну из сторон — закрывает
  твин-коллизии («лут»→"ken", "key"→«лун», «дсд»→"lcl") и токены вида
  "i'm"→«шэь». Слова с реальными уликами гиббериша (фетч-класс, ";bpym")
  сохраняют облачное восстановление.
- Словарное правило теперь требует, чтобы typed-сторона выглядела гиббершем по
  n-gram: правильно набранные «лут»→"ken", «луты»→"kens", "iban"→«шифт»,
  «дсд»→"lcl" больше не конвертируются локально. Локальные ложные конверсии на
  17 169 правильно набранных словах: **17 → 0** (воспроизведено независимым
  агентом, в обоих режимах чувствительности).

### Короткие слова не конвертировались (BUG-1) — исправлено
- Минимальная длина слова теперь считается по буквам ЦЕЛЕВОГО слова, а не
  латинского твина: «это» (`'nj`), «уже» (`e;t`), «без» (`,tp`), «бок» (`,jr`),
  «жук», «жар» — конвертируются при floor 3. Слова с буквами б/ю/ж/э/х на
  пунктуационных клавишах больше не наказываются.
- «же», «бы», «юг», «хэш», «эхо» перестали быть неконвертируемыми даже хоткеем
  (гейт «≥2 букв» тоже мерил не ту сторону).

### Кейс «yt описано» (BUG-7) — ретро-доконверсия
- Когда слово уверенно конвертируется по границе, предыдущее короткое слово
  (та же исходная раскладка, один пробел, свежее, твин — частотное/выученное
  слово, например «yt»→«не») доконвертируется тем же вбросом. Строгие условия:
  каретка не двигалась, слово не ветировано, длина ниже floor.

### Live-режим («на лету») — переработан (BUG-3, BUG-4, BUG-5)
- Синхронный спелчек полностью убран с пути каждой клавиши: латентность
  decide() **310 мкс avg / 6,7 мс max → 32 мкс / 80 мкс** — класс лагов
  «tap timeout → теряются клавиши» устранён.
- Словарное правило больше не применяется к префиксам: набор «буфер» не
  ломается на «буфе»→",eat", «путей» — на «путе»→"gent". Мид-ворд ложные
  срабатывания на правильном наборе: **19 → 0** (RU), **34 → 0** (EN).
- Конверсия применяется после паузы в наборе (дебаунс 250 мс), а не между
  клавишами — гонка синтетических backspace'ов с быстрым набором закрыта.
- После live-конверсии слово ПРОДОЛЖАЕТСЯ в буфере в целевой раскладке —
  «хвост» больше не оценивается как отдельное огрызок-слово.

### Обучение на отказах (BUG-8) — Backspace больше не ядовит
- Голый Backspace после конверсии больше НЕ считается отказом (это обычно
  редактирование — стереть пробел, поставить запятую). Из-за него слова
  навсегда переставали конвертироваться (negative-записи копились в learned.json).
- Новые сигналы отказа: **⌘Z** в окне 20 с после автоконверсии (сессионное вето
  сразу + постоянный запрет после 2 событий) и **удалил-и-перепечатал** то же
  слово. ⇧⌘Z (redo) и ⌘Z в другом приложении не считаются.

### Прочее
- Enter/Tab больше не триггерят конверсию (BUG-9): в чатах Enter уже отправил
  сообщение — «призрачный» текст в пустом поле больше не появляется; слово
  остаётся доступным для ручного хоткея.
- После автоконверсии хоткей снова работает: история слов не стирается (BUG-6),
  повторный хоткей откатывает конверсию (слово хранится в целевой раскладке).
- Конверсия выделения ориентируется по скрипту самого текста, а не по текущей
  системной раскладке (BUG-11).
- Ручная смена раскладки посреди слова больше не стирает историю предыдущего
  слова (BUG-12).

### Адверсариальная проверка (6 независимых агентов, задача — опровергнуть фиксы)
Ядро устояло: 0 локальных FP на 17 169 слов воспроизведено независимо; live-путь
доказан инструментированным прогоном (0 обращений к спелчеку на 24 809 оценок);
5 338 смоделированных live-конверсий — 0 съеденных символов. Найденные дефекты
исправлены в этом же релизе:
- **Ретро-фикс портил текст при нарушенной геометрии** (двойной пробел
  «yt␣␣word» → «yне описано»; стёртый Backspace'ом пробел → съедание символа
  из предыдущего текста). Теперь любой «голый» разделитель или Backspace при
  пустом буфере отвязывает завершённое слово от автоматических правок.
- **BUG-12 был починен не в том месте**: историю стирал наблюдатель смены
  раскладки (`hardReset`), а не буфер. Теперь ручная смена раскладки сохраняет
  историю (хоткей работает), но отвязывает её от ретро-фикса.
- **Окно ⌘Z не закрывалось** кликом/стрелками/Enter — чужой undo мог быть
  приписан конверсии; теперь любое такое событие снимает окно. **⌃Z** больше
  не считается undo (различаем Command и Control).
- **Live-конверсия могла сработать после выключения** тумблера (внутри
  250 мс дебаунса) — добавлена перепроверка настроек.
- Твин-коллизии и "i'm"-класс в облачном пути — закрыты вторым AI-гейтом
  (см. выше). Удалён недостижимый код (второе словарное правило aggressive),
  убрано мёртвое поле истории.

### Регрессионные тесты
- `--selftest`: +12 кейсов на исправленные баги + 9 проверок state-machine
  буфера + 8 проверок облачного гейта (всё зелёное, exit 0). Пробник аудита —
  fp/fn/prefix/floor/bench свипы — в скретчпаде сессии аудита.

### Известные компромиссы и остаточные риски
- 3-буквенные некураторские/невыученные слова с «чистым» латинским твином
  («лут»/«мяч»-класс) автоматически не конвертируются вовсе (ни локально, ни
  через AI) — только хоткей или обучение: на такой длине сигналов недостаточно,
  и раньше этот механизм давал «лут»→"ken".
- 2-буквенные слова («не», «он») по-прежнему ниже floor 3 — их закрывает
  ретро-доконверсия по контексту следующего слова.
- Ретро-фикс может конвертировать НАМЕРЕННО набранное короткое слово перед
  кривым («vs jgbcfyj» → «мы описано»): локального сигнала различить «vs» и
  «yt» нет. Риск ограничен: правка видима, ⌘Z учит движок, вето запоминается.
- Узкая полоса: правильно набранные слова с n-gram 0.01–0.05, чей твин —
  реальное слово, по-прежнему решаются облачной моделью (доп. защита — ⌘Z).
- Детекция ⌘Z — по физической позиции QWERTY-Z; на Dvorak может не сработать
  (теряется только сигнал обучения, не конверсия).
