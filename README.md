# ghbdtn

Локальная утилита для macOS, которая **автоматически исправляет текст, набранный не в той раскладке** — в духе Punto Switcher, но полностью офлайн и с опциональным облачным ИИ-слоем.

> Живёт в строке меню рядом с часами. Нажатия анализируются **в памяти** и никуда не отправляются — кроме случая, когда вы сами включите облачный ИИ-слой.

---

## Что умеет

1. **Конвертация «абракадабры».** Набрали `ghbdtn` вместо `привет` (или `руддщ` вместо `hello`) — приложение это замечает по завершении слова (или на лету, после короткой паузы набора — режим настраивается), стирает и допечатывает правильный вариант, а заодно переключает системную раскладку. Если после конвертированного слова уверенно конвертируется следующее, короткое слово перед ним («yt» → «не») доисправляется автоматически.
2. **Определяет язык само, без хардкода пар.** Читает *все включённые в системе раскладки* и для каждого слова сравнивает, в какой из них текст выглядит «настоящим». Работает не только для ru↔en — любая пара раскладок, для которых у macOS есть словарь проверки орфографии, поддерживается автоматически.
3. **Живёт в трее** (menu bar) — иконка клавиатуры рядом с часами, меню с переключателем, недавними конвертациями и настройками.
4. **Страница настроек** — шесть вкладок: Общие, Детекция, Словарь (просмотр и правка выученных слов), Хоткеи, ИИ-слой, Голос.
5. **Облачный ИИ-слой (опция).** OpenAI-совместимый провайдер подключается для *спорных* слов и исправлений целого текста. Ключ хранится в Keychain, не в настройках.
6. **Голосовой ввод (Whisper).** Ставите курсор в любое текстовое поле, жмёте хоткей (`⌃⌥V`) или пункт «Диктовка» в меню трея — появляется плавающая панель с живой волной, таймером и двумя кнопками-символами (⏹ отменить, ✓ распознать). Распознанный текст вставляется прямо в позицию курсора. Движок на выбор: **локальный whisper.cpp** (полностью офлайн, Metal-ускорение) или **облачный** (любой OpenAI-совместимый `/audio/transcriptions`: OpenAI, Groq).
7. **Учится на ваших исправлениях.** Дожали слово ручным хоткеем (`⌃⌥Space`) — после пары повторов оно начинает конвертироваться само. Так движок осваивает имена, сленг и редкие термины, которых нет в словарях. И наоборот: если авто-режим сконвертировал лишнее, нажмите ⌘Z (или сотрите и наберите как было) — конверсия этого слова блокируется сразу на сессию, а после пары раз — насовсем. Всё локально, в `~/Library/Application Support/Ghbdtn/learned.json`.
8. **Обновляется сам.** Раз в сутки проверяет GitHub Releases; новая версия — уведомление и пункт «⬆ Обновить» в меню: один клик, скачивание, проверка подписи, перезапуск. Права выдавать заново не нужно.

## Установка

### Обычный способ — скачать готовый .dmg

1. Откройте [**Releases**](../../releases/latest) и скачайте `ghbdtn-X.Y.Z.dmg`.
2. Откройте образ и перетащите **ghbdtn.app** в папку **Программы** (ярлык рядом).
3. **Первый запуск** (один раз): приложение подписано без платного сертификата
   Apple, поэтому macOS его карантинит («не удаётся открыть» / «повреждено»).
   Самый быстрый способ снять карантин — одна команда в Терминале:

   ```bash
   xattr -dr com.apple.quarantine /Applications/ghbdtn.app
   ```

   После неё приложение открывается обычным двойным кликом. Без Терминала:
   правый клик по ghbdtn в Программах → «Открыть» → в диалоге ещё раз «Открыть»
   (если диалог без кнопки — Системные настройки → Конфиденциальность и
   безопасность → внизу «Открыть всё равно»).

### Единственный ручной шаг — выдать права

macOS не даёт приложению выдать себе права само. После установки:

1. **Системные настройки → Конфиденциальность и безопасность → Универсальный
   доступ** → включить **ghbdtn**. Нужно, чтобы видеть нажатия клавиш и вставлять
   исправления. Без этого **не конвертируется ничего** — это первое, что стоит
   проверить, если «не работает».
2. Для голосового ввода — разрешить **Микрофон** при первом запросе.

Проверка: наберите `ghbdtn` в любом поле → должно стать `привет`.
Живёт в строке меню (иконка клавиатуры). Хоткеи: `⌃⌥Space` — ручная конвертация,
`⌃⌥V` — диктовка.

### Обновления — сами

Приложение раз в сутки проверяет новые версии на GitHub. Когда выходит новая,
в меню трея появляется пункт **«⬆ Обновить до X»** и приходит уведомление —
один клик, и приложение скачает релиз, проверит подпись, обновит себя и
перезапустится. Права заново выдавать не нужно. Ручная проверка — пункт
«Проверить обновления» в меню; отключить автопроверку можно в Настройках.

### Удаление

Перетащите ghbdtn из Программ в Корзину. Данные (выученные слова, модели
Whisper) лежат в `~/Library/Application Support/Ghbdtn` — удалите папку, если
не планируете возвращаться. При установке из исходников есть `./uninstall.sh`
(с `--purge` — стереть и данные).

## Для разработчиков

### Установка из исходников (одна команда)

```bash
git clone <repo-url> ghbdtn && cd ghbdtn
./install.sh
```

Скрипт сам: проверит macOS (нужна **13.3+**); если нет **Command Line Tools** —
**запустит их установку и дождётся** (нужно один раз нажать «Install» в системном
окне, перезапускать install.sh не нужно); скачает прибандленный whisper.cpp
(~48 МБ, с проверкой контрольной суммы), соберёт релиз, подпишет и **установит
`ghbdtn.app` в `/Applications`**, запустит и откроет нужную панель настроек.
Бонус локальной сборки: приложение не попадает в карантин Gatekeeper — шаг с
`xattr` не нужен.

Флаги установщика:

| Флаг | Что делает |
|---|---|
| _(без флагов)_ | ad-hoc подпись — прав хватает; но после каждого обновления `Универсальный доступ` нужно выдать заново |
| `--stable-signing` | создаёт постоянную самоподписанную личность → **разрешение переживает обновления** (один раз попросит «Always Allow» в связке ключей) |
| `--clean` | установить **без зашитых выученных слов** (`seed-learned.json`) — чистое приложение, знающее только доменные словари. По умолчанию выученные слова включены |
| `--no-launch` | не запускать приложение и не открывать настройки после установки |

Обновление из исходников: `git pull && ./install.sh` (или та же кнопка
самообновления в меню — она не трогает dev-сборки вне /Applications).

### Сборка без установки

```bash
./build.sh run        # собрать release, упаковать в ghbdtn.app и запустить (без копии в /Applications)
./build.sh            # только собрать + упаковать
./build.sh debug      # debug-сборка
./tools/fetch-whisper.sh && swift build   # просто скомпилировать
.build/release/Ghbdtn --selftest          # регрессионные проверки: детекция, обучение,
                                          # длина слова, гейты облака, буфер, голос
.build/release/Ghbdtn --updatecheck       # проверка апдейтера (GHBDTN_FAKE_VERSION=0.4.0 — сценарий «есть обновление»)
./tools/make-dist.sh                      # собрать релизный .dmg в dist/
```

> **Про подпись.** `build.sh` подписывает стабильной самоподписанной личностью
> «Ghbdtn Local Signing», если она есть (создаётся `tools/setup-signing.sh`) —
> тогда разрешение `Универсального доступа` держится между пересборками. Иначе
> откатывается на ad-hoc: разрешение слетает при каждой пересборке (её designated
> requirement — cdhash бинарника, а он меняется каждый билд). Для публичного
> распространения нужен Developer ID + нотаризация.

### Выученные слова

Слова, которые вы учите вручную (`⌃⌥Space`), лежат локально в
`~/Library/Application Support/Ghbdtn/learned.json` (правятся во вкладке
«Словарь»). В репозиторий зашит небольшой стартовый набор
`Sources/Ghbdtn/Resources/seed-learned.json`, который приложение подмешивает
при старте; не хотите его — ставьте с `--clean`.

## Как работает детекция

Ключевая идея: **не хранить таблицу «й=q, ц=w…»**, а выводить соответствие
раскладок из самой системы.

1. **`KeyTranslator`** через `UCKeyTranslate` для каждой включённой раскладки
   строит карту `символ → физическая клавиша` и обратно. Это работает для любой
   раскладки (Русская, Русская-ПК, Украинская, и т.д.) без ручных таблиц.
2. Для набранного слова (последовательности *физических клавиш*) считаются его
   интерпретации во всех раскладках. `ghbdtn` под английской = `ghbdtn`, под
   русской = `привет`.
3. **`LanguageScorer`** оценивает каждую интерпретацию несколькими сигналами:
   - **курируемые списки частых слов** (ru/en/uk) **плюс выученные из ваших
     исправлений** (`LearnedStore`) — самый надёжный сигнал, он же страхует от
     редких ложных срабатываний системного словаря;
   - **`NSSpellChecker`** — точная проверка по словарю языка (авторитетно для
     целых слов; поддерживает все языки, для которых у macOS есть словарь);
   - **символьная 4-граммная модель** (`NgramModel`) — статистическая оценка
     «похоже ли это на настоящий текст языка» для всего, чего словарь не знает:
     имён, сленга, редких слов и **недопечатанных частей слова**. Модели
     обучены офлайн на реальных корпусах (Leipzig Corpora: новости + веб)
     **вперемешку с курируемыми доменными терминами** (музыка, программирование,
     3D, аудио — чтобы заимствования вроде «сэмпл», «шейдер», «сайдчейн»
     конвертировались) со сглаживанием Кнезера–Нея и зашиты в приложение
     (~0,5 МБ на язык, оценка
     слова ≈ 1,5 мкс). Сырые вероятности разных языков несравнимы, поэтому
     оценка калибруется в **перцентиль среди настоящих слов этого языка** —
     и уже перцентили двух интерпретаций сравниваются между собой.
4. **`Decider`** конвертирует, только когда «переставленная» интерпретация
   заметно более похожа на настоящий язык, чем набранная. Слово, которое уже
   является корректным словом своего языка, никогда не трогается. Для n-gram
   слоя пороги подобраны офлайн-перебором по полному словарю корпуса до **нуля
   ложных срабатываний** (`tools/eval_thresholds.py`) и зависят от
   чувствительности (осторожный / сбалансированный / агрессивный).
5. Спорные случаи можно отдавать **облачному ИИ** (если включён) — ему уходит
   только одно слово с вариантами интерпретаций. Вердикт модели проходит жёсткие
   локальные гейты: он обязан совпадать с раскладочным «твином» тех же клавиш
   (модель не может «перевести» слово — «фетч» никогда не станет "fetch") и
   отклоняется, если ни один локальный сигнал не может судить ни набранное,
   ни результат.

Всегда есть **ручной хоткей** (`⌃⌥Space` по умолчанию): конвертирует выделенный
текст или последнее слово принудительно — на случай, если авто-режим ошибся или
промолчал.

### Переобучение n-gram моделей

Модели в `Sources/Ghbdtn/Resources/Models/*.bin` воспроизводимы. Языки — en, ru,
uk (задаются в `train_ngram.CORPORA` + `ngram_lm.ALPHABETS`). В обучение
подмешиваются курируемые доменные словники из `tools/domain-corpora/{lang}.txt`:

```bash
python3 tools/train_ngram.py        # скачает корпуса (~100 МБ, кэшируются) и обучит все языки
#   --domain-weight N   вес доменных терминов (псевдо-счётчик, дефолт 50)
python3 tools/eval_thresholds.py    # перебор порогов: граница нуля ложных срабатываний
#   → перенесите её в Settings.ngramThresholds, если модель менялась
swift build && .build/release/Ghbdtn --selftest   # полный набор регрессионных проверок
```

Чтобы добавить термины — допишите слова в `tools/domain-corpora/{lang}.txt` и
переобучите. Добавить язык — alphabet + CORPORA + TOKEN_RE + строка в
`LanguageScorer.seedModels`. Формат сериализации и эталонный скорер — в
`tools/ngram_lm.py`; Swift-реализация (`Core/NgramModel.swift`) зеркалит его
бит-в-бит, поэтому пороги, подобранные в Python, переносятся без пересчёта.

> **Важно для коротких слов.** Символьная модель не оценивает слова короче 4
> букв и объективно редкие (напр. «пэд», «пэды») — их закрывает слой обучения
> (`⌃⌥Space` ×2), курируемые списки частых слов и ретро-доконверсия по контексту
> (короткое слово перед уверенно сконвертированным соседом), а не n-gram.
> Минимальная длина слова (Настройки → Детекция) считается по буквам
> **целевого** слова, поэтому «бок» (`,jr`) и «это» (`'nj`) — трёхбуквенные,
> сколько бы букв ни было в их латинском написании.

## Голосовой ввод (Whisper)

Диктовка в **любое** текстовое поле любого приложения:

1. Ставите курсор, жмёте `⌃⌥V` (настраивается) или «Диктовка» в меню трея.
2. Появляется плавающая панель: живая волна с микрофона, таймер `мм:сс`,
   ⏹ (отменить) и ✓ (распознать). Панель **не забирает фокус** — курсор
   остаётся в вашем поле (неактивирующая `NSPanel`, показ без активации
   приложения).
3. По ✓ аудио уходит в выбранный движок, результат вставляется в позицию
   курсора: короткие фразы — синтетическим вводом, длинные — атомарной
   вставкой через буфер обмена (буфер сохраняется и восстанавливается).

**Движки** (Настройки → Голос):

- **Локальный** — whisper.cpp (прибандленный XCFramework, Metal на Apple
  Silicon). Модель скачивается при первом включении с HuggingFace в
  `~/Library/Application Support/Ghbdtn/Models/` с проверкой SHA-256; аудио
  никуда не отправляется. По умолчанию `large-v3-turbo` (q5, ~574 МБ) —
  лучший баланс русский/английский; есть tiny/base/small для экономии места.
  Модель держится в памяти между диктовками (загрузка модели, а не
  распознавание, — главная задержка).
- **Облачный** — любой OpenAI-совместимый `POST /audio/transcriptions`:
  OpenAI (`gpt-4o-transcribe`) или Groq (`whisper-large-v3-turbo` — те же
  веса, что локально, почти бесплатно). Пресеты в настройках. Ключ — в
  Keychain; если пусто, берётся ключ ИИ-слоя. Аудио уходит провайдеру —
  движок строго опциональный.

Кнопка **«Проверить сейчас»** в настройках показывает результат распознавания
прямо в окне (без вставки) — удобно сравнить локальный и облачный движки и
переключиться, если локальное качество не устраивает.

Почему нет пункта в контекстном меню по правому клику: macOS не позволяет
стороннему приложению добавлять пункты в контекстные меню чужих приложений —
каждое приложение строит своё меню само. Хоткей + меню трея — честный
системный эквивалент (так же работают superwhisper и встроенная диктовка).

## Приватность и безопасность

- Приложение **не в песочнице** (глобальный event tap несовместим с App Sandbox) —
  распространять через Developer ID, не через Mac App Store.
- Нажатия клавиш **не логируются и не сохраняются**. Буфер держит только текущее
  слово и очищается на границах, кликах, смене приложения и при защищённом вводе
  (пароли — `IsSecureEventInputEnabled`).
- В «Детекции» можно исключить приложения (терминалы, игры, менеджеры паролей);
  по умолчанию исключены Terminal, iTerm2 и Связка ключей.
- API-ключ ИИ хранится только в Keychain.
- Автопроверка обновлений раз в сутки запрашивает у `api.github.com` номер
  последнего релиза — никаких данных о вводе или системе не передаётся;
  отключается в Настройках → Общие.

## Ограничения и что дальше

- Детекция намеренно консервативна: пороги и гейты выставлены так, чтобы
  корректный текст не портился никогда, поэтому часть редких слов вне словаря
  не конвертируется автоматически. Отдельный класс — короткие (3 буквы) слова,
  которых нет ни в частотных списках, ни в выученных («лут», «мяч» и их
  латинские твины): на такой длине сигналов недостаточно, их конвертирует
  только ручной хоткей — а после двух раз они выучиваются и идут сами.
- Слова, завершённые **Enter/Tab**, не конвертируются автоматически — осознанно:
  в чате Enter уже отправил сообщение, и поздняя правка печатала бы «призрачный»
  текст в пустое поле. Ручной хоткей работает и для них.
- Встроенные n-gram модели покрывают **ru, en, uk** (с доменными терминами);
  для украинского нужно включить украинскую раскладку в macOS, иначе она не
  станет кандидатом при конвертации. Для других языков детекция опирается на
  словарь macOS (свою модель можно добавить в
  `~/Library/Application Support/Ghbdtn/Models/ngram-<язык>.bin`).
- Голосовой ввод: локальная модель большая (рекомендуемая — ~574 МБ), качество
  русского у маленьких моделей заметно хуже; в шуме/с акцентом облачный
  `gpt-4o-transcribe` точнее локального. Пункта в контекстном меню чужих
  приложений нет и не будет — macOS это запрещает (см. «Голосовой ввод»).
- Иконка приложения пока системная (SF Symbol в трее).
- Замена текста делается синтетическими нажатиями; в приложениях с агрессивным
  автодополнением возможны краевые случаи — тогда удобнее ручной хоткей на
  выделенном тексте (он идёт через буфер обмена и восстанавливает его).

## Устранение неполадок

- **«Не удаётся открыть» / «приложение повреждено»** при первом запуске
  скачанного .dmg — это карантин Gatekeeper (подпись без платного Apple
  Developer ID). Одна команда в Терминале решает:
  `xattr -dr com.apple.quarantine /Applications/ghbdtn.app` — либо правый клик
  → «Открыть». Нужно один раз; самообновления в карантин не попадают.
- **Ничего не конвертируется** (даже `ghbdtn` → `привет`). Почти всегда это
  права: **Универсальный доступ** не выдан или слетел. Настройки →
  Конфиденциальность → Универсальный доступ → снять и снова поставить галочку у
  `ghbdtn` (или удалить «–» и добавить «+» заново). Это первое, что нужно
  проверять — а не логику детекции.
- **После обновления перестало работать** (касается сборки из исходников с
  ad-hoc подписью: она меняется при каждой пересборке, и macOS сбрасывает
  грант). Лечится переустановкой прав, а навсегда — установкой со стабильной
  подписью: `./install.sh --stable-signing`. Релизные .dmg и встроенное
  самообновление подписаны стабильной личностью — у них грант не слетает.
- **Нет `swift` / сборка падает.** `./install.sh` сам ставит Command Line Tools
  (нажмите «Install» в системном окне и подождите — скрипт продолжит сам). Если
  собираете вручную через `./build.sh` — сначала `xcode-select --install`.
- **Украинский (или другой язык) не конвертируется.** Включите его раскладку в
  Настройки → Клавиатура → Источники ввода — движок берёт цели только из
  включённых раскладок.
- **Короткое/редкое слово не конвертируется** («пэд», имя, сленг). Это by design
  (см. врезку про короткие слова в разделе про детекцию) — дожмите ручным хоткеем `⌃⌥Space` дважды,
  дальше пойдёт само.
- **Диктовка не слышит микрофон.** Настройки → Конфиденциальность → Микрофон →
  включить `ghbdtn`.
