# ghbdtn

Локальная утилита для macOS, которая **автоматически исправляет текст, набранный не в той раскладке** — в духе Punto Switcher, но полностью офлайн и с опциональным облачным ИИ-слоем.

Название `ghbdtn` — это «привет», набранное в английской раскладке. Как раз та боль, которую приложение лечит.

> Живёт в строке меню рядом с часами. Никаких окон, пока они не нужны. Нажатия анализируются **в памяти** и никуда не отправляются — кроме случая, когда вы сами включите облачный ИИ-слой (и туда уходит лишь одно спорное слово, не весь текст).

---

## Что умеет

1. **Конвертация «абракадабры» на лету.** Набрали `ghbdtn` вместо `привет` (или `руддщ` вместо `hello`) — приложение это замечает по завершении слова, стирает и допечатывает правильный вариант, а заодно переключает системную раскладку.
2. **Определяет язык само, без хардкода пар.** Читает *все включённые в системе раскладки* и для каждого слова сравнивает, в какой из них текст выглядит «настоящим». Работает не только для ru↔en — любая пара раскладок, для которых у macOS есть словарь проверки орфографии, поддерживается автоматически.
3. **Живёт в трее** (menu bar) — иконка клавиатуры рядом с часами, меню с переключателем, недавними конвертациями и настройками.
4. **Страница настроек** — пять вкладок: Общие, Детекция, Хоткеи, ИИ-слой, Голос.
5. **Облачный ИИ-слой (опция).** OpenAI-совместимый провайдер подключается для *спорных* слов. Ключ хранится в Keychain, не в настройках.
6. **Задел под голосовой ввод (Whisper).** Хоткей, права на микрофон и выбор движка (локальный/облачный) уже есть; сама транскрипция намеренно не реализована — это следующий шаг.

## Сборка и запуск

Нужны только **Command Line Tools** (полный Xcode не требуется), macOS 13+.

```bash
cd keysw
./build.sh run        # собрать release, упаковать в ghbdtn.app и запустить
# или:
./build.sh            # только собрать + упаковать
./build.sh debug      # debug-сборка
swift build           # просто скомпилировать без бандла
```

При первом запуске появится окно-онбординг с просьбой выдать доступ в
**Системные настройки → Конфиденциальность и безопасность → Универсальный доступ**.
Это нужно, чтобы видеть нажатия клавиш и вставлять исправления. После включения
галочки окно продолжит работу само.

> **Про подпись.** Скрипт подписывает бандл ad-hoc. macOS привязывает разрешение
> Универсального доступа к подписи, поэтому после каждой пересборки разрешение
> нужно выдать заново (macOS покажет его как новое приложение). Для «настоящего»
> распространения подпишите Developer ID — тогда разрешение сохранится между
> обновлениями.

## Как работает детекция

Ключевая идея: **не хранить таблицу «й=q, ц=w…»**, а выводить соответствие
раскладок из самой системы.

1. **`KeyTranslator`** через `UCKeyTranslate` для каждой включённой раскладки
   строит карту `символ → физическая клавиша` и обратно. Это работает для любой
   раскладки (Русская, Русская-ПК, украинская, и т.д.) без ручных таблиц.
2. Для набранного слова (последовательности *физических клавиш*) считаются его
   интерпретации во всех раскладках. `ghbdtn` под английской = `ghbdtn`, под
   русской = `привет`.
3. **`LanguageScorer`** оценивает каждую интерпретацию несколькими сигналами:
   - **курируемые списки частых слов** (ru/en) — самый надёжный сигнал, он же
     страхует от редких ложных срабатываний системного словаря;
   - **`NSSpellChecker`** — точная проверка по словарю языка (авторитетно для
     целых слов; поддерживает все языки, для которых у macOS есть словарь);
   - **символьная 4-граммная модель** (`NgramModel`) — статистическая оценка
     «похоже ли это на настоящий текст языка» для всего, чего словарь не знает:
     имён, сленга, редких слов и **недопечатанных частей слова**. Модели
     обучены офлайн на реальных корпусах (Leipzig Corpora: новости + веб) со
     сглаживанием Кнезера–Нея и зашиты в приложение (~0,5 МБ на язык, оценка
     слова ≈ 1,5 мкс). Сырые вероятности разных языков несравнимы, поэтому
     оценка калибруется в **перцентиль среди настоящих слов этого языка** —
     и уже перцентили двух интерпретаций сравниваются между собой.
4. **`Decider`** конвертирует, только когда «переставленная» интерпретация
   заметно более похожа на настоящий язык, чем набранная. Слово, которое уже
   является корректным словом своего языка, никогда не трогается. Для n-gram
   слоя пороги подобраны офлайн-перебором по полному словарю корпуса до **нуля
   ложных срабатываний** (`tools/eval_thresholds.py`) и зависят от
   чувствительности (осторожный / сбалансированный / агрессивный).
5. Спорные случаи можно отдавать **облачному ИИ** (если включён) — ему уходит
   только одно слово с вариантами интерпретаций.

Всегда есть **ручной хоткей** (`⌃⌥Space` по умолчанию): конвертирует выделенный
текст или последнее слово принудительно — на случай, если авто-режим ошибся или
промолчал.

### Переобучение n-gram моделей

Модели в `Sources/Ghbdtn/Resources/Models/*.bin` воспроизводимы:

```bash
python3 tools/train_ngram.py        # скачает корпуса (~100 МБ, кэшируются) и обучит
python3 tools/eval_thresholds.py    # перебор порогов: граница нуля ложных срабатываний
swift build && .build/debug/Ghbdtn --selftest   # 18 кейсов детекции + замер задержки
```

Формат сериализации и эталонный скорер — в `tools/ngram_lm.py`; Swift-реализация
(`Core/NgramModel.swift`) зеркалит его бит-в-бит, поэтому пороги, подобранные в
Python, переносятся в приложение без пересчёта.

```

│   └── SpeechEngine.swift     Задел под Whisper (локальный/облачный) + хоткей
├── Settings/
│   ├── Settings.swift         Все настройки (UserDefaults + Keychain для ключа)
│   ├── SettingsView.swift     SwiftUI-настройки (5 вкладок)
```

## Приватность и безопасность

- Приложение **не в песочнице** (глобальный event tap несовместим с App Sandbox) —
  распространять через Developer ID, не через Mac App Store.
- Нажатия клавиш **не логируются и не сохраняются**. Буфер держит только текущее
  слово и очищается на границах, кликах, смене приложения и при защищённом вводе
  (пароли — `IsSecureEventInputEnabled`).
- В «Детекции» можно исключить приложения (терминалы, игры, менеджеры паролей);
  по умолчанию исключены Terminal, iTerm2 и Связка ключей.
- API-ключ ИИ хранится только в Keychain.

## Ограничения и что дальше

- N-gram слой намеренно консервативен: пороги выставлены так, чтобы корректный
  текст не портился никогда, поэтому часть редких слов вне словаря он пропустит
  (например, заимствования с нетипичной для языка статистикой букв — «смузи») —
  для таких случаев есть ручной хоткей и опциональный ИИ-слой.
- Встроенные n-gram модели покрывают пары ru↔en; для других языков детекция
  опирается на словарь macOS (модель можно добавить в
  `~/Library/Application Support/Ghbdtn/Models/ngram-<язык>.bin`).
- Голосовой ввод (Whisper) — **задел**: движок не подключён.
- Иконка приложения пока системная (SF Symbol в трее).
- Замена текста делается синтетическими нажатиями; в приложениях с агрессивным
  автодополнением возможны краевые случаи — тогда удобнее ручной хоткей на
  выделенном тексте (он идёт через буфер обмена и восстанавливает его).
